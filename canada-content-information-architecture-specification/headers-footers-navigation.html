Headers, footers and navigation for Canada.ca

From Treasury Board of Canada Secretariat

Document navigation

  • Previous: Conceptual overview of user navigation and key templates for Canada.ca

  • Next: Usage of the Canada.ca design

Navigation through Canada.ca is supported by the use of 3 navigation systems:

  1. global header and footer menus,
  2. local menus
  3. in-page navigation mechanisms.

All navigation components are designed to adapt to a variety of screen sizes to ensure accessibility and ease of use on a wide range of Internet-enabled devices.

Global header and footer navigation

Global navigation menus encourage content discovery through the primary and secondary navigation pathways of the site’s user navigation model:

  • Primary: by themes and topics (using the global navigation menu in the header at the top of each page)
  • Secondary: by institution or organization (using the departments and agencies landing page via the footer)
Local navigation

Local navigation menus are optional navigation components used to offer visitors easy access to sibling destination pages adjacent to their current location in the site’s user navigation model. In large screen views, local navigation appears as a left menu; on small screens, this component slides to the bottom of the page’s content area.

In-page navigation

In-page navigation mechanisms handle a person’s movement on the destination page or throughout the task. A number of in-page navigation patterns are defined in this section to assist with different types of content. New patterns are under development.

Global header navigation

Mandatory

The top tasks from all Government of Canada websites are organized into topics and then grouped into the 15 main themes of Canada.ca, represented in the header.

The themes themselves are prioritized by traffic, placing the 7 most popular themes in the most visible part of the global navigation bar, and the remaining 8 themes under a single grouping entitled “More services.”

Objective

The purpose of the themes and topics navigation menu is to help people discover or locate destination tasks and content available in the deeper parts of the site’s information architecture.

This global navigation menu allows access to all GC content and is available to people across Canada.ca to facilitate movement, findability and access.

When to use the global header navigation

The global navigation menu must be used on all Canada.ca pages, with the following exceptions:

  • may be optional for:
    • transactional scenarios
    • campaign and promotional events pages
  • menu items (themes) may not be altered without approval from Treasury Board Secretariat
  • sub-menu items (topics) may not be altered without approval from the Theme Lead department accountable for that theme

How to implement

The global navigation menu must always adhere to the themes and topics approach described in this document. The labels that are used in the global navigation menu should correspond with the Topic tree for Canada.ca.

The global header and footer cannot be altered and must remain consistent, wherever it is used.

On large screens

  • Clicking or hovering on one of the 7 main themes reveals a submenu containing the top topics for that theme:
    • these submenu items directly correspond with the top topics listed on the theme page, and must be listed in the same order
    • no additional links may be added to the submenus
    • the last item in this submenu is always a link to more information, leading to the landing page for the theme itself
  • Clicking or hovering on “More services” reveals a submenu listing the remaining 8 themes:
    • the last item in the “More services” submenu is always a link to “All services,” which leads people to a page listing all of the site’s 15 themes presented as topics with descriptions and full titles

On smaller screens

  • The global menu is relocated to a user-triggered menu panel optimized for tablets and smart phone devices, also known as a “hamburger menu”
    • this panel may only be used for global navigation; never use a second panel of this type on any given page
  • The site search, language toggle and footer menus are also included in this mobile panel
  • The hidden submenus under each of the top-level menu items are revealed on clicking

Other header elements

Mandatory

In addition to the global navigation menu, the header includes additional branding and functional elements. Generally speaking, all elements of the header must be used on Canada.ca to ensure a consistent experience.

When to use the other header elements

  • The government of Canada signature is always mandatory.
  • Remaining global header elements must be used on all Canada.ca pages, with the following exceptions:
    • global header elements may be optional for:
      • transactional scenarios
      • campaign and promotional events pages

Government of Canada signature

The Government of Canada signature as specified in Federal Identity Program (FIP) policy instruments is prominently featured in the global header area across Canada.ca.

Objective

The purpose of the Government of Canada signature is to provide an easily identifiable graphic brand. This signature also serves as a universal link back to the site’s home page at www.canada.ca.

How to implement the Government of Canada signature

  • The Government of Canada signature is presented as an image file and must be formatted according to FIP design specifications
  • The image is provided as a Scalable Vector Graphics (SVG) file, configured to scale automatically according to screen size to allow for responsive design
  • The image is hyperlinked to the Canada.ca home page

Site search box

The ability to search all GC content in Canada.ca is supported by a simple search field, which is provided in the global header used across the site.

Objective

The search feature provides an alternative method to discover content that can be used by visitors to directly locate any content contained on Canada.ca. It allows people to bypass the government-wide navigation layer completely by submitting keywords that the system will match with relevant GC content.

How to implement the Site search box

  • Basic search consists of 2 elements: a keyword text field and a search button to initiate the query
  • Basic search is accessible through the global header area, across all pages of the site. On smaller screens, it is located inside the mobile slide-out menu panel triggered by the button in the top right corner of the header
  • The basic search function allows for the use of Boolean operators in the text field to submit more refined or specific queries to the search engine
  • Basic search also includes type-ahead (auto-complete) functionality, so that as a person begins typing a query in the search box, suggested words will automatically appear below
  • Exceptionally, the search box may be omitted on (see search results pages), where the search box is already included in the content area of the page. It may also be omitted on (see transactional pages).

Language toggle link

All GC content under the Canada.ca domain is available in both official languages, at a minimum. A language toggle link in the global site header gives people access to site content in either French or English.

Certain sections of Canada.ca may be made available in additional languages. These languages would be accessed through the language toggle link as well. Policy guidance on the use of additional languages appears in the Policy on Communications and Federal Identity of the Government of Canada.

Objective

The language toggle link always leads people to the equivalent page in the other official language, ensuring that people are able to switch between languages without losing track of where they were before switching.

How to implement the language toggle link

  • The language toggle link consists of only one element—a simple text link indicating either “English” or “Français”
  • The link is presented in the top right corner of the global header area, across all pages of the site
  • On smaller screens, it is housed at the top of the mobile-friendly slide-out menu panel triggered by the button in the header

Breadcrumb trail

A horizontal series of links known as a “breadcrumb trail” is presented just beneath the global themes and topics navigation menu in the header across all pages of the site.

The links in the breadcrumb trail are not intended to reflect the path by which the visitor arrived at the current location. Rather, the links always represent the location of the current page as it stands in relation to the site’s user navigation model.

Objective

The purpose of the breadcrumb trail is to reinforce a visitor’s current location in the Canada.ca user navigation model, by indicating the location of the current page in relation to its parent in the IA. It also provides users with a simple method for moving upwards through the site structure, in cases where they wish to access higher-level sections without being forced to return to the site’s home page or search.

How to implement the breadcrumb trail

  • The breadcrumb trail begins with “Home” (a link to the home page in each language of Canada.ca) and ends one level above the visitor’s current page on the site.

    For example, if a person is on the “Tips for healthy eating” page on the Health Theme, the breadcrumb trail will be “Home arrow pointing right Health arrow pointing right Food and nutrition arrow pointing right Healthy eating”.

  • On mobile, only 2 links should be displayed in the breadcrumb trail. When intermediate links have been removed, an ellipsis (…) must be used to signal that levels are being skipped in the breadcrumb trail

    For example, if a user is on the “Tips for healthy eating” page on the Health Theme on mobile, the breadcrumb trail will be “Home arrow pointing right … arrow pointing right Healthy eating” not “Home arrow pointing right Health arrow pointing right Food and nutrition arrow pointing right Healthy eating”.

  • The current page is never displayed at the end of the breadcrumb trail.
  • Arrows are used as separators, as per W3C recommendations.
  • Longer page titles are to be shortened where possible, to help reduce the horizontal space required to display the trail. Each page or section name on the Canada.ca site should be associated with a shortened version of the name, for the purposes of breadcrumb and other navigation systems. For example, “Business and industry” is shortened to “Business” in the trail.

Main content footer

Mandatory

The main content footer appears after the main content of the page, and above the global site footer. Only the following components are allowed in this area:

  • Report a problem
  • Share this page
  • Date modified

Report a problem on this page

The Report a problem component allows people to provide feedback via a form for any problems that may have happened. This component is intended to collect and address any issues for the page on which it resides.

Objectives

Use this component to enable page-level feedback. Common web page content issues are presented for a person to select using checkboxes. If none of these options are applicable, they may select “Other”.

When to use Report a problem on this page

The report a problem component is mandatory for all pages on Canada.ca.

How to implement Report a problem on this page

  • The component is presented as a button using WET style for secondary actions
  • The label reads: “Report a problem on this page”
  • Placement: positioned directly below the main content area, above the Date modified component, left-aligned
  • When clicked, the button triggers a form which expands immediately below the button
    • page components that follow are pushed down to accommodate the form
    • when clicked a second time the form will collapse
  • The form header reads “Please select all that apply:”
  • The form presents 6 checkbox options and a submit button styled as a primary action, as shown in Figure 6
  • When a checkbox is selected, a multiline text input field appears to allow a person to effectively enter, edit and review their input
    • these fields are labeled: “Provide more details (optional):”
  • The form includes a hyperlink to a Privacy statement, which appears in a dismissible overlay when clicked
  • The submit button is labeled “Submit”
  • When a person hits the “Submit” button, the form is submitted and replaced by a thank you message
    • the thank you message can be collapsed by clicking the “Report a problem with this page” button.
    • the form must then be reset so that people may submit any other problems they encounter on that page

Share this page

The Share this page component is intended to allow people to share the content of the page to other users.

Objectives

The Share this page component allows people to share the current web page via a selection of sharing platforms and delivery services.

When to use Share this page

  • This component is optional
  • In transactional scenarios, the button must share a link to the first page of the transactional process

How to implement Share this page

  • The component is presented as a button using WET style for secondary actions
  • The label reads: “Share this page”, preceded by the sharing icon in the defined icon set included in WET
  • Placement: this component is right-aligned, positioned opposite the Report a problem component.
  • When clicked, a dismissible overlay is triggered:
    • the overlay is labeled “Share this page” and includes buttons for a selection of sharing platforms and delivery services, and the message “No endorsement of any products or services is expressed or implied.”
    • on large screens, the buttons appear in 2 columns
    • on small screens, a person is presented with a single column of buttons
  • Within the overlay, the button for each sharing platform or delivery service uses WET style for secondary actions
  • The label for each button within the overlay includes the name of the sharing platform or delivery service, preceded by a relevant logo or icon
  • When clicked, each button within the overlay triggers a link to that platform or service, including appropriate parameters, such as the URL or title of the page being shared
    • these links open in new browser tabs or windows where feasible
  • Available sharing platforms and delivery services will be reviewed by the Principal Publisher on a regular and as-needed basis

Date modified

The date modified component must indicate how recently the content of a web page has been updated.

Objectives

Use the date modified component to provide a person with the date that the content was last updated.

When to use the Date modified

This component:

  • is mandatory on all pages with the exception of transactional scenarios, where the component can be replaced by a “Version number” if desired
  • should generally only be updated when significant content changes occur
  • should generally not be updated for corrections to typographical errors or changes to the underlying HTML code
  • can be updated manually to signify that the content has been fully reviewed and is still valid as of that date

How to implement the Date modified

  • This component always starts with the words “Date modified:” followed by a date.
  • The date appears using the common alphanumeric method, for example, October 4, 2016.
  • Placement: left-aligned, below the Report a problem button.

Global footer navigation

Mandatory

The global (site-wide) footer at the bottom of each web page contains links to Government of Canada corporate content, and to Government of Canada content repositories.

Also included in the footer are utility links encouraging people to contact the Government of Canada, provide feedback on their Canada.ca experience, and explore GC social media channels.

Objective

The purpose of footer navigation menus is to help people discover government-related content contained in the deeper parts of the site’s user navigation model. The footer menus also provide direct, centralized access to GC-wide tasks (for example, Contact us, News, Open government).

When to use the global footer navigation

  • The global footer is mandatory on all pages of Canada.ca
  • The footer may be modified for:
    • transactional scenarios
    • campaign and promotional events pages

How to implement the global footer navigation

The footer navigation menu is accessible at the bottom of all pages of the site, in both large and small screen views. On smaller screens, it is also located inside the mobile-friendly slide-out menu panel triggered by the button in the top right corner of the header.

Structure

Global footer links must remain the same throughout the site.

Government-wide tasks and government links have been prioritized according to demand, derived from search logs and web traffic data. They appear in the following order:

  • Contact us
  • Departments and agencies
  • Public service and military
  • News
  • Treaties, laws and regulations
  • Government-wide reporting
  • Prime Minister
  • How government works
  • Open government

Utility links appear after links to government-wide tasks and government links, in the following order:

  • Social media
  • Mobile applications
  • About Canada.ca
  • Terms and conditions
  • Privacy

Other footer elements

The Canada wordmark is prominently featured in the global footer across all pages of Canada.ca. This serves to reinforce the brand and provide an additional cue to people that they are reading content from the Government of Canada.

On smaller screens, a “Top of page” link is presented in the footer area, along with the Canada Wordmark.

Reduced header and footer

Conditional

Objective

  • Provides flexibility for specific situations in which full application of the global header and footer impede task success, while maintaining overall consistency in the experience

When to use the reduced header and footer

The reduced header and footer can only be used in the following situations:

  • Transactional scenarios
  • Campaign pages
  • Promotional events pages

How to implement the reduced header and footer

On pages using the reduced header and footer page template, the Canada.ca header is reduced to the following elements:

  • Government of Canada FIP signature, linked to the Canada.ca home page
  • language toggle link, linked to the equivalent page in the other official language
  • breadcrumb trail, with the main landing page for the GC ad site at the first level after Home.

All other parts of the global header are removed.

The Canada.ca footer is reduced to the following elements:

  • 3 footer links, contextualized with appropriate
    • Contact information
    • Terms and conditions
    • Privacy notice
  • Canada wordmark
  • “Top of page” link (small screen sizes only)

All other parts of the global footer are removed.

On small screens, the remaining elements of the global header and footer are accessible through a user-triggered menu panel optimized for tablets and smart phone devices, also known as a “hamburger menu”. This panel may only be used for global navigation; never use a second panel of this type on any given page.

Transactional scenarios

Transactional scenarios are pages where people are engaged in a transactional process—where allowing them to navigate away from the current page would interrupt the intended flow and result in errors, loss of data or accidental termination of the session.

As such, the reduced version header and footer may be used for transactions. However, given the highly variable nature of different transactional scenarios, some additional elements of the global header and footer may also be included or modified, as follows.

Local navigation menus

Local navigation menus are selectively used to establish connections between related groups of destinations and tasks within a given branch of the Canada.ca user navigation model.

Objective

Local navigation menus support user-access to sibling destination pages adjacent to their current location in the site’s user navigation model, allowing people to jump between tasks without having to move back up the hierarchy to select a new destination.

Section menus always relate to a person’s current position within the site hierarchy. As a person moves deeper into the site, the menu changes to show the sibling pages adjacent to a person’s current position in the user navigation model. As such, these menus “follow” a person as they go deeper into the destination layer of Canada.ca.

How and when to use local navigation menus

Use and positioning of local navigation menus

  • Use of local navigation menus is optional
    • they are implemented at the discretion of content owners
  • Local navigation menus must not be used on navigation pages (for example, on theme or topic pages, institutional profile pages)
    • they are permitted only on destination pages, below the government-wide navigation layer
  • These menus may be used in cases where people are likely to benefit from being able to move laterally between closely related tasks (for example, radar maps and satellite maps), or between sub-pages of a task destination (for example, “get a study permit” and “extend your study permit”)
    • they must be avoided where the tasks or destinations are not closely related for example, “product recalls” and “emergency preparedness”)
  • Do not use local navigation menus when a suitable in-page navigation mechanism is already available, such as:
    • table of contents patterns for long documents
    • step-by-step patterns, such as on advanced service initiation pages
    • tabbed interface patterns for showing related aspects of destination content
  • When a local navigation menu is implemented on a page, the menu must be used on all other pages that appear on the menu
    • this ensures that the menu does not disappear when a person makes a selection
  • Local navigation menus consist of
    • a clickable header link, identifying the immediate parent of the current page
    • clickable links to sibling pages that permit visitors to move across pages at the same level as the current page
  • Local navigation menus are displayed using the WET section menu component:
    • on large screens, they are located on the left side of the page, in the content area
    • on smaller screens, the menu appears at the bottom of the page content area, immediately above the global footer

Contents of local navigation menus

  • Local navigation menus use only one level of links, allowing people to move laterally between sibling tasks. These menus must never be used to offer links leading deeper into the current task or destination, that may be handled by in-page content links and navigation systems (see the section on in-page navigation).

A person’s current position in the menu is marked by a coloured box or background on the corresponding menu item. This current page item must not be linked; instead, it must be non-clickable text.

Local navigation menus – Interim solution

To facilitate the onboarding process, the local navigation menu component will allow for a second level of links to be added to the menu. This interim version of the menu will be available to all departments for the duration of the onboarding stage, after which institutions will be expected to phase out all second-level links over the course of the optimization stage of the project.

This additional level of menu links will be used to display the offspring of the current destination page, to accommodate downward movement deeper into the current task.

In-page navigation

Canada.ca makes use of a variety of in-page navigation patterns to facilitate movement through destination and task content. The patterns have been developed to support different use-case scenarios and content types.

All types of in-page navigation mechanisms are intended for use inside the content area of a given page, to complement the global and local menu systems. The patterns may only be used inside task or destination areas of the site, below the navigation layer.

Do not apply multiple in-page navigation patterns on individual pages or groups of pages. When designing task or destination content, choose the most appropriate option, and use it consistently.

In-page table of contents

The in-page table of contents pattern is a list of links that lead to subsections of the same page. This pattern is intended for long documents that exist as a single page.

Objective

Use the in-page table of contents pattern to:

  • provide an outline for the content of a single page that has multiple subsections
  • provide an alternative method to jump to specific content on the page, without using the scroll bar

Use this pattern when:

  • excessive scrolling is required to view all of the content on the page
  • the page is divided into subsections, each with its own headings

Do not use this pattern when the document exists across several web pages. Instead, use the multi-page table of contents pattern.

How and when to use an in-page table of contents

  • Always position the in-page table of contents pattern at the start of the page, before the substantive content of the page begins
  • Label the pattern “On this page” to distinguish it from other page content and to provide consistency throughout the site
  • Present links using list elements—bullets for an unordered list, numbers for an ordered list
  • Use standard linking styles to ensure that the links present as clickable elements
  • Present the in-page table of contents aligned to the left, as a vertical list, as opposed to using a column layout

Ordered multi-page navigation (step-by-step)

The ordered multi-page navigation pattern:

  • is a set of links to a group of several pages intended to be read in a preferred order
  • includes previous/next links to guide a person through the page set in the preferred order

Objective

Use the ordered multi-page navigation pattern to provide:

  • an outline of content that spans multiple pages and has a default or preferred order (for example, guides, service initiations, sequential instructions)
  • a method of jumping to any page within the sequence, bypassing the default order of the pages

Do not use this pattern when:

  • the content exists as a single page requiring scrolling to view all the subsections (use the In-page table of contents pattern)
  • the content exists as a set of multiple pages that does not have a default or preferred order
  • the default order of the content must not be circumvented that is when the preferred order is crucial and people cannot be permitted to move between pages in a different order

How and when to use ordered multi-page navigation

  • Always position the ordered multi-page navigation pattern at the start of the page, before the substantive content of the page begins
  • Ensure that the order and position of the page links are identical on each page of the set
  • Number the step-by-step links to indicate the preferred order. Not all links necessarily need to be numbered, if they represent pages/steps that are not part of the sequence
  • Highlight the link for the current page. Do not highlight with the same colour used on primary action buttons
  • Position previous/next links immediately below the substantive content on the page
  • Accompany the previous/next labels with arrow icons to reinforce the concept
  • Position previous/next links consistently (for example, “previous” on the left, “next” on the right)
  • On the first page of the document, omit the “previous” link; on the last page of the document, omit the “next” link

Tabbed interface

The tabbed interface pattern separates related content into different panes that are viewable one at a time when the respective tab is clicked.

Objective

Use the tabbed interface pattern to group and present different aspects of content that are closely related, such as:

  • Geographic places, where the tabs can be used to present content relating to a place’s location, demographics, economy, etc.
  • Product catalogues, where the tabs can be used to present consistent facets of each product in the catalogue, such as specifications, features, etc.

Do not use the tabbed interface pattern when:

  • content must be presented in a specific sequential order (instead, use a step-by-step pattern)
  • content needs to be compared and contrasted, which would lead to repeated switching between tabs (instead, use tables or other patterns that allow all content to be presented at once)
  • content is lengthy or detailed, because of space limitations in the tabbed interface
  • content presentation includes only 2 tabs (for greater efficiency, use subheadings)

How and when to use a tabbed interface

  • Use defined presentation classes for the tabbed interface included in WET
  • Ensure that tab headers are clear and concise; avoid lengthy labels
  • When repeating the tabbed interface across a set of related pages (for example, on country pages), always ensure the tab headers are presented consistently
  • Make the default active tab first on page load; but another tab may be used as the active tab on page load if required
